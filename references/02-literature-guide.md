# 文献全流程指南：从检索到综述

> 适用场景：当你不知道去哪里找文献、不知道怎么搜、不知道高亮文献后怎么读和怎么记笔记时，读这份指南。

---

## 1. 文献去哪找

### 1.1 中文数据库

| 数据库 | 地址 | 特点 | 适合什么 |
|--------|------|------|---------|
| **知网 (CNKI)** | cnki.net | 中文最全，论文/期刊/学位论文都有 | 中文文献首选 |
| **万方** | wanfangdata.com.cn | 与知网互补 | 知网找不到时补搜 |
| **百度学术** | xueshu.baidu.com | 中文+英文混搜，免费 | 快速粗搜 |
| **维普** | cqvip.com | 中文期刊 | 第三种选择 |

> 🎓 **学校福利：** 知网/万方/维普通过学校图书馆网站访问，校内 IP 直接免费下载。校外用学校 VPN。

### 1.2 英文数据库（CS/AI 方向特别重要）

| 数据库 | 地址 | 特点 | 什么时候用 |
|--------|------|------|-----------|
| **Google Scholar** | scholar.google.com | 最方便，搜索质量好，显示引用数 | 日常搜索首选 |
| **Semantic Scholar** | semanticscholar.org | AI 驱动，有引用图谱、高影响力论文标记 | AI/CS 论文特别好用 |

## 2. 怎么搜——关键词策略

### 2.1 三个步骤

```
研究问题 → 拆出核心概念 → 找每个概念的同义词 → 组合搜索
```

### 2.2 完整示例

**研究问题：** "基于大语言模型的编程学习辅助系统的设计与评估"

#### Step 1：拆出核心概念
- 概念1：大语言模型
- 概念2：编程学习 / 编程教育
- 概念3：系统设计 / 效果评估

#### Step 2：列同义词（中英文）

| 核心概念 | 中文关键词 | 英文关键词 |
|---------|-----------|-----------|
| 大语言模型 | 大语言模型, 大模型, LLM, ChatGPT, GPT-4, 代码大模型, 生成式AI | large language model, LLM, ChatGPT, GPT, GPT-4, code generation model, Codex, Copilot |
| 编程学习 | 编程学习, 编程教育, 编程教学, 程序设计, 计算机教育, CS1 | programming education, CS education, CS1, programming learning, coding education, novice programmer, introductory programming |
| 系统/效果 | 智能辅导系统, 学习辅助, 教学辅助工具, 学习效果, 编程能力 | intelligent tutoring system, learning assistant, programming assistant, AI tutor, learning outcome, learning performance, code quality |

#### Step 3：组合搜索

**在知网专业检索中：**
```
SU = ('大语言模型' + 'LLM' + 'ChatGPT' + '生成式AI') * ('编程' + '程序设计') * ('学习' + '教育' + '教学')
```

**在 Google Scholar 中：**
```
("large language model" OR "ChatGPT" OR "LLM") AND ("programming education" OR "CS1" OR "novice programmer") AND ("learning" OR "assistant" OR "tool")
```

**在 Semantic Scholar 中：**
```
large language model | LLM | ChatGPT
programming education | CS education | novice programmers
```

### 2.3 增加文献量的技巧

如果搜出来的文献太少（<10篇）：

1. **减少 AND 条件** —— 先用一个大概念搜，再人工筛选
2. **扩大时间范围** —— 从近 3 年放宽到近 5 年
3. **去掉次要概念** —— 只保留 1-2 个核心关键词
4. **查引用文献的参考文献** —— 找到一篇相关的 → 看它引了什么 → 追溯

### 2.4 减少文献量的技巧

如果搜出来几百上千篇：

1. **加 AND 条件** —— 把次要概念加进去
2. **缩小时间范围** —— 只看近 2 年
3. **只看核心来源** —— 中文限定 CSSCI/核心期刊，英文限定 CCF 推荐会议/期刊
4. **按引用量排序** —— 先看引用最多的前 20 篇

## 3. 怎么读——三遍阅读法

### 3.1 三遍法总览

```
第一遍：5-10分钟  → 决定"要不要继续读"
第二遍：15-30分钟 → 理解"做了什么"和"主要结果"
第三遍：1-2小时   → 彻底理解、能复现思路
```

大多数论文只需要读到第二遍。

### 3.2 第一遍：标题 + 摘要 + 结论

**具体操作：**
1. 读标题 → 能不能看懂主题？
2. 读摘要 → 作者研究了什么问题？用了什么方法？主要结果是什么？
3. 跳到结论（论文最后一两段）→ 作者自己怎么总结？

**最重要的决策：** 读完这三部分后，问自己：
- 这篇论文和我的研究相关吗？
- 如果不相关 → **果断扔掉**，不要犹豫（一篇文章淘汰 60-70% 是正常的）
- 如果相关 → 进入第二遍

### 3.3 第二遍：图表 + 关键段落

**具体操作：**
1. 翻到图表 → 看图 X 和表 X → 每张图在说什么？
2. 读引言最后一段 → 通常包含"本文贡献"
3. 扫方法部分的关键子节 → 数据怎么来的、模型/系统怎么设计的
4. 看结果部分的关键数字 → 核心指标是多少、对比差多少

**第二遍读完你应该能回答：** 这篇论文做了什么？怎么做的？主要结果是什么？我能不能用它的方法或结论？

### 3.4 第三遍：精读

只有 1/3 左右的论文需要精读——通常是：
- 你的方法的直接竞争对手
- 你准备在其基础上改进的论文
- 领域中最重要的经典文献

**精读要点：**
- 逐段理解论证逻辑
- 搞清楚方法的每一个细节（参数、配置、数据集）
- 理解为什么用这个指标、为什么这个参数设这个值
- 思考：如果我来做，我会怎么改进？

## 4. 怎么做笔记——文献矩阵法

### 4.1 矩阵模板

读完一篇、填一行。用你习惯的方式（Excel / Notion / 甚至聊天记录）：

| 列名 | 填什么 | 示例 |
|------|--------|------|
| **文献** | 作者 + 年份 + 标题关键词 | Zhang 2024, "CodeTutor: A LLM-based..." |
| **研究问题** | 一句话：作者要解决什么 | 如何让 LLM 提供个性化的编程错误提示 |
| **理论框架** | 用了什么理论/概念（社科/交叉学科尤其重要） | 认知负荷理论 (Cognitive Load Theory) |
| **方法** | 怎么做的：实验？系统开发？调查？ | 开发了一个 VS Code 插件 + 用户实验 (n=80) |
| **关键发现** | 主要数据/结论 | 实验组编程作业正确率提升 18%，但学习时间没有显著减少 |
| **局限性** | 作者自己承认的问题 | 样本规模较小，实验周期仅 4 周 |
| **与我的关系** | 我为什么记这篇 | 可以参考其评价指标体系；我想改进其缺乏长期追踪的局限 |

### 4.2 "理论框架"列为什么重要

你学的是信息管理与信息系统（计算机+管理交叉），这一列对交叉学科尤其关键。

**例子：** 关于"为什么学生使用 AI 工具"，不同文献可能用完全不同的理论解释：

- Zhang (2023) 用 **技术接受模型 (TAM)** → 强调"感知有用性"和"感知易用性"
- Li (2024) 用 **自我决定理论 (SDT)** → 强调"内在动机"和"自主性"
- Wang (2023) 用 **UTAUT** → 强调"社会影响"和"便利条件"

写文献综述时，你按理论流派分组（"TAM视角的研究发现...；SDT视角的研究则强调..."），而不是按作者罗列。

### 4.3 边读边写综述

当你填满 10+ 行矩阵后，文献综述的骨架就出来了：

1. 按"研究问题"分组 → 识别出几个研究方向
2. 按"理论框架"分组 → 识别出几种理论视角
3. 按"方法"对比 → 识别出主流方法 vs 新方法
4. 看"局限性"列 → 这就是你的研究要填补的空白

## 5. 怎么判断一篇文献值不值得读

### 5.1 快速质量判断

- ✅ 发表在 CCF 推荐会议/期刊（计算机领域）
- ✅ 发表在 CSSCI/核心期刊（中文）
- ✅ 引用量 > 50（英文）/ > 30（中文）= 领域基础文献
- ✅ 近 3 年发表 + 引用量增长快 = 前沿热点
- ⚠️ 发表在普通期刊/不知名会议 → 注意判断质量
- ❌ 无出处的工作论文/博客 → 可以看思路，但不宜作为主要引用

### 5.2 相关性判断

| 情况 | 处理 |
|------|------|
| 同一问题、不同方法 → | 精读！这是你的竞争对手 |
| 同一方法、不同问题 → | 读第二遍，看看有没有可借鉴的 |
| 问题无关但方法相关 → | 读第一遍，记一下方法思路 |
| 完全无关 → | 果断扔掉 |

## 6. 从文献矩阵到文献综述

### 6.1 常见错误

```
❌ "文献堆砌式"综述：
"张三 (2023) 研究了...李四 (2024) 提出了...Wang (2023) 发现..."
→ 读者看完不知道这些文献之间有什么关系
```

### 6.2 正确写法：主题式组织

```
✅ "主题式"综述：
"大语言模型在编程教育中的应用已成为一个活跃研究领域。在效果评估方面，
多项研究一致发现 LLM 辅助学习能显著提升编程成绩（张三 2023；Wang 2023）。
具体而言，Zhang (2024) 的元分析报告了约 18% 的平均提升效果。
然而，在工具形态上，现有研究存在分歧——部分采用对话式界面（李四 2024），
另一部分采用代码补全模式（Liu 2023）...

值得注意的是，大多数现有研究仅关注短期效果（4-8 周实验周期），
对长期编程能力发展的影响尚不清楚——这正是本研究的切入点。"
```

**关键原则：**
- 按主题分组（不要按作者分组）
- 点出共识（"多项研究一致发现..."）
- 指出争议（"现有研究存在分歧..."）
- 点明空白（"对...的影响尚不清楚"）
- 衔接你的研究（"这正是本研究的切入点"）
